To download the tar file of coreutils, we run the command 

wget ftp://ftp.gnu.org/gnu/coreutils/coreutils-8.29.tar.xz

to download the signature, we run the command 

wget 
ftp://ftp.gnu.org/gnu/coreutils/coreutils-8.29.tar.xz.sig

to download the keyring we run the command 

wget https://ftp.gnu.org/gnu/gnu-keyring.gpg

to verify its signature, we run the command 

gpg --verify --keyring 
./gnu-keyring.gpg coreutils-8.29.tar.xz.sig

gpg responds that the signature is good but it is not
certified with a trusted signature!
This is because we have not signed it with our own key,
because we do not have one.
It does not reflect on the safety of the package itself.

to extract its files, we run 

tar xvfJ coreutils-8.29.tar.xz

we then create the installation folder for the coreutils

mkdir coreutilsInstall

Then we enter the coreutils-8.29 directory and configure 
the prefix with our new installation directory

./configure --prefix=/u/cs/ugrad/hakan/coreutilsInstall

we then run the following commands to install the package
into our installation directory.

make 
make install

There were no problems with the installation. 

to reproduce the bug, we create a shell script named la
with the following contents.

#!/bin/sh
exec ls -a "$@"

We then must change permissions so we can run la 

chmod +x la 

we run ./la -A in our working directory to observe the bug.
it produces this output:

.  ..  coreutils-8.29  coreutils-8.29.tar.xz  
coreutils-8.29.tar.xz.sig  coreutilsInstall
gnu-keyring.gpg  la  lab3.txt

the la -A command is supposed to produce the above output
but without the . and ..

We try the ls -a -A command from coreutilsInstall that
la -A translates to. It produces the same result

./coreutilsInstall/bin/ls -a -A

.  ..  coreutils-8.29  coreutils-8.29.tar.xz  
coreutils-8.29.tar.xz.sig  coreutilsInstall
gnu-keyring.gpg  la  lab3.txt

We also try the ls -a -A command that la -A translates
to. It produces the same output, confirming that this 
is a bug with the coreutils.

./coreutils-8.29/src/lsls -a -A

.  ..  coreutils-8.29  coreutils-8.29.tar.xz  
coreutils-8.29.tar.xz.sig  coreutilsInstall
gnu-keyring.gpg  la  lab3.txt

To get the patch we get the patch file posted in message
10 of bug report 30963

wget "https://debbugs.gnu.org/cgi/bugreport.cgi?msg=10;
bug=30963;filename=0001-ls-A-now-overrides-a.patch;att=1"


we rename it to patchFile for legibility

mv bugreport.cgi\?msg\=10\;bug\=30963\;filename
\=0001-ls-A-now-overrides-a.patch\;att\=1 patchFile

we move it to coreutils-8.29 and apply this patch with
the command 

patch -p1 < patchFile

The -p1 option removes one of the leading slashes from
each filename in patchFile.

we then run make but do not follow with make install

make

autoreconf -f -i
